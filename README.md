# 发酵罐控制项目技术文档

## 1. 项目简介

这是一个基于 **STM32H750VBT6** 的发酵罐控制固件项目，用来完成发酵过程中的多路采集与执行器控制。  
这个项目主要围绕以下几类功能展开：

- 温度采集与加热控制
- pH 采集与酸碱泵调节
- 搅拌电机控制与转速读取
- 流量计读取
- 尾气参数读取
- 溶氧采集
- 上位机串口指令接入

当前采用的是 **裸机 + HAL + DMA + 轮询调度** 的方式运行，后续将升级为FREErtos。

## 2. 使用的技术与工具

### 2.1 芯片与固件基础

- MCU: `STM32H750VBTx`
- 固件库: `STM32Cube FW_H7 V1.12.1`
- 驱动层: `STM32 HAL` + `CMSIS`
- 代码语言: `C`
- 启动文件: `startup_stm32h750vbtx.s`
- 链接脚本:
  - `STM32H750VBTX_FLASH.ld`
  - `STM32H750VBTX_RAM.ld`

### 2.2 工程与开发环境

项目里同时保留了两套工程：

- `STM32CubeMX / STM32CubeIDE` 工程
  - 配置文件: `CONTROL_1.ioc`
  - Eclipse 工程文件: `.project`、`.cproject`、`.mxproject`
  - `Debug/` 目录下有 GCC 构建产物和 makefile
- `Keil MDK-ARM` 工程
  - `MDK-ARM/CONTROL_1.uvprojx`
  - `MDK-ARM/CONTROL_1.uvoptx`

从 `CONTROL_1.ioc` 看，CubeMX 里配置的编译器是 **GCC**；但项目也同步维护了 **Keil** 工程，说明你当时大概率两边都能编译下载。

## 3. 项目总体架构

源码主要分为三层：

- `Core/Src`、`Core/Inc`
  - 主业务逻辑、外设初始化、控制算法、各传感器/执行器模块
- `Drivers`
  - STM32 HAL 与 CMSIS 官方驱动
- `MDK-ARM`
  - Keil 工程文件与构建输出

业务模块基本都是按设备拆分的，每个设备一个 `.c/.h`：

- `ph.c`：pH 采集与酸碱调节
- `pump.c`：酸泵、碱泵控制
- `fed.c`：补料泵控制预留
- `stir.c`：搅拌电机控制与读速
- `flower.c`：流量计读取与目标流量设置
- `endgas.c`：尾气分析模块读取
- `pt100.c`：PT100 温度模块读取
- `ntc_control.c`：温控主逻辑
- `pid.c`：PID 与自整定
- `ads8688.c`：外扩 ADC 采集溶氧
- `relay.c`：继电器控制
- `pwm2.c`：PWM 输出控制预留

## 4. 当前已启用的核心功能

根据 `main.c` 当前实际调用情况，已经接入主流程的功能有：

### 4.1 温度控制

- 通过 `PT100` 模块读取温度
- `pt100.c` 中做了简单的滑动平均滤波
- `ntc_control.c` 中实现了三阶段温控：
  - 预热阶段
  - PID 自整定阶段
  - PID 正常控制阶段
- 最终通过 `relay.c` 控制继电器，实现加热开关控制

当前代码中的关键参数：

- 目标温度 `SETPOINT = 30.0°C`
- 预热接近阈值 `0.5°C`
- 预热超时 `60s`
- 自整定时长 `300s`

温控不是模拟量连续输出，而是根据 PID 结果和升温趋势，转换为 **继电器开/关控制**。

### 4.2 pH 采集与酸碱调节

- `ph.c` 通过串口读取 pH 设备数据
- 采用类似 Modbus RTU 的报文格式
- 读取到 pH 后，根据设定值自动启动酸泵或碱泵
- 酸泵、碱泵控制由 `pump.c` 完成

当前代码中的关键参数：

- 目标 pH：`6.0`
- 死区：`±0.2`
- 单次泵运行时间：`1500 ms`
- 冷却时间：代码值为 `18000 ms`

控制逻辑是典型的 **死区式开关调节**，不是 PID 调 pH。

### 4.3 搅拌电机控制

- `stir.c` 负责搅拌电机的串口控制
- 上电后会发送使能命令与速度模式命令
- 支持设置转速 `Set_Stir_Speed()`
- 支持读取当前搅拌转速 `Read_Stir_Speed()`
- 还支持通过 `USART1` 接收上位机下发的二进制指令，再转发为电机控制命令

从代码看，搅拌控制协议也是 **Modbus 风格串口协议**。

### 4.4 流量计读取

- `flower.c` 中实现了流量计读数与目标流量设置
- 当前主流程使用的是 `Task_Flow()` 周期读取
- 解析结果包含：
  - 当前流量 `current_flow3`
  - 目标流量 `target_flow3`
- 单位为 `LPM`

从报文看，流量计设备地址是 `0x03`。

### 4.5 尾气数据读取

- `endgas.c` 通过 `UART4` 与尾气模块通信
- 周期性发送读取命令并解析返回值
- 当前解析出的量包括：
  - 尾气温度
  - CO2 浓度
  - O2 浓度

这部分目前偏调试风格，串口打印很多原始数据，说明当时你应该还在边联调边完善。

### 4.6 溶氧采集

- `ads8688.c` 使用 `SPI4` 驱动外部 ADC `ADS8688`
- 当前主循环会周期调用 `ADS8688_ReadOxygen(4)`
- 代码中已实现：
  - ADC 原始值读取
  - 电压换算
  - 4~20mA 电流换算
  - DO 原始值换算
  - 两点标定
  - 简单均值滤波

这说明你的溶氧通道不是直接进 MCU ADC，而是走 **外部高精度 ADC + 模拟量变送器** 方案。

### 4.7 上位机串口通信

- `USART1` 被用作调试/上位机接口
- `printf` 被重定向到了 `USART1 DMA`
- 上位机可以发送 8 字节控制报文
- 固件收到后会调用 `Process_Stir_Command()`，用于远程控制搅拌速度

所以这个项目已经具备一个基础的 **串口上位机交互入口**。

## 5. 外设与接口分配

按当前代码推断，接口分配大致如下：

| 外设 | 作用 | 关键参数 |
| --- | --- | --- |
| `USART1` | 上位机通信、日志输出、搅拌命令入口 | `115200` |
| `USART3` | pH 模块 | `38400` |
| `USART6` | 酸泵/碱泵/补料泵 | `115200` |
| `UART4` | 尾气分析模块 | `9600` |
| `UART7` | 搅拌电机、流量计 | `19200` |
| `UART8` | PT100 温度模块 | `115200` |
| `SPI4` | ADS8688 外扩 ADC | 主机模式 |
| `TIM1 CH4` | PWM 输出预留 | `PE14` |
| `TIM4` | 基础定时器 | 系统计时辅助 |
| `PB0` | 继电器输出 | 加热控制 |

与 SPI 相关的 GPIO：

- `PE2` -> `SPI4_SCK`
- `PE5` -> `SPI4_MISO`
- `PE6` -> `SPI4_MOSI`
- `PE4` -> ADS8688 `CS`
- `PC13` -> ADS8688 `RST`
- `PE3` -> ADS8688 `DAISY/控制脚`

## 6. 主循环调度方式

项目没有使用任务系统，核心逻辑在 `main()` 的 `while(1)` 里通过 `HAL_GetTick()` 分时执行：

- `USART1` 收到控制命令时立即处理
- 每 `3000 ms` 执行一次 `Task_PH()`
- 每 `1000 ms` 执行一次 `Endgas_Task()`
- 每 `500 ms` 执行一次 `NTC_Control_Update()`
- 每 `2000 ms` 执行一次 `ADS8688_ReadOxygen(4)`
- 每 `2000 ms` 执行一次 `Task_Flow()` 和搅拌速度读取

通信层大量使用了：

- `HAL_UART_Transmit_DMA()`
- `HAL_UARTEx_ReceiveToIdle_DMA()`

说明你当时已经把 **DMA + 空闲中断收包** 用进来了，这对于多串口设备并行通信是比较实用的方案。

## 7. 控制策略总结

这个项目目前实现的控制策略可以概括为：

- 温度：`PT100 + 滤波 + PID/自整定 + 继电器开关控制`
- pH：`周期采样 + 阈值判断 + 酸泵/碱泵定时补偿`
- 搅拌：`串口设置转速 + 周期读回实际转速`
- 溶氧：`模拟量采集与换算`，当前更偏监测，闭环控制尚未真正启用
- 流量：`周期读取当前流量与目标流量`
- 尾气：`周期采集 O2 / CO2 / 温度`

从整体上看，这是一个典型的 **发酵罐多参数监测 + 多执行器联动控制** 固件。
